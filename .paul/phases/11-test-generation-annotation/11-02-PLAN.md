<objective>
## Goal
Add the annotation workflow to the Test stage — users can approve, reject, or edit individual questions inline, and perform bulk approve/reject actions across a test set.

## Purpose
Before running experiments, users need to review and curate their test questions. This plan adds the annotation UI so users can approve good questions, reject bad ones, edit reference answers, and bulk-process remaining questions. The summary bar (from Plan 11-01) updates in real time to show annotation progress.

## Output
- Inline annotation actions on each question card (approve, reject, edit)
- Edit mode with editable reference answer and notes field
- Bulk action bar (approve selected, reject selected, approve all pending, reject all pending)
- Summary bar refreshes after annotation actions
- All wired to existing PATCH and POST /bulk endpoints
</objective>

## Relevant API Endpoints
**Single Question Annotation:**
- PATCH /api/projects/{id}/test-sets/{test_set_id}/questions/{question_id}
  - Body: { status: "approved"|"rejected"|"edited", user_edited_answer?: string, user_notes?: string }
  - If status="edited", user_edited_answer is REQUIRED (422 if missing)
  - Returns: updated question object

**Bulk Annotation:**
- POST /api/projects/{id}/test-sets/{test_set_id}/questions/bulk
  - Body: { action: "approve"|"reject"|"approve_all"|"reject_all", question_ids?: number[] }
  - "approve"/"reject": requires question_ids list
  - "approve_all"/"reject_all": applies to all pending questions (no question_ids needed)
  - Returns: { updated_count: number }
</context>

## AC-1: Individual Question Annotation
```gherkin
Given a question card in the question list
When the user clicks "Approve" on a question
Then PATCH is called with status="approved"
And the question's status badge updates to green "approved"
And the summary bar refreshes to reflect the change
When the user clicks "Reject" on a question
Then PATCH is called with status="rejected"
And the question's status badge updates to red "rejected"
And the summary bar refreshes
And annotation buttons are disabled while the request is in progress
```

## AC-2: Edit Annotation
```gherkin
Given a question card in the question list
When the user clicks "Edit" on a question
Then the card expands into edit mode showing:
  - The original reference answer (read-only, for context)
  - An editable textarea pre-filled with the current reference answer (or user_edited_answer if previously edited)
  - A notes textarea (optional)
  - "Save" and "Cancel" buttons
When the user modifies the answer and clicks "Save"
Then PATCH is called with status="edited", user_edited_answer, and user_notes
And the card returns to display mode with updated "edited" badge (blue)
And the summary bar refreshes
When the user clicks "Cancel"
Then edit mode closes without changes
```

## AC-3: Bulk Actions
```gherkin
Given the question list with a selection mechanism
When the user selects multiple questions via checkboxes
Then a bulk action bar appears showing: "N selected" with "Approve Selected" and "Reject Selected" buttons
When the user clicks "Approve Selected"
Then POST /bulk is called with action="approve" and question_ids of selected
And all selected questions update to "approved"
And selection is cleared and summary refreshes
When the user clicks "Approve All Pending" (always visible in bulk bar)
Then POST /bulk is called with action="approve_all"
And all pending questions become "approved"
And summary refreshes
And bulk action buttons are disabled while the request is in progress
```
