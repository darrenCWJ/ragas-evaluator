## Goal
Build the test set generation form, test set list with management, and question list view — the first half of the Test stage UI.

## Purpose
Users need to generate test Q&A sets from their chunked documents before they can run experiments. This plan creates the generation workflow (persona config, size settings, trigger) and lets users view/manage their test sets and browse individual questions.

## Output
- Test set generation form (chunk config selector, size, persona settings, generate trigger)
- Test set list with stats (total questions, status counts, delete capability)
- Question list view (browse questions in a test set, status badges, filtering)
- All wired to existing FastAPI endpoints
</objective>

## Relevant API Endpoints
**Test Sets:**
- POST /api/projects/{id}/test-sets — create (body: TestSetCreate with chunk_config_id, name, testset_size, num_personas, custom_personas, use_personas)
- GET /api/projects/{id}/test-sets — list (returns test_sets array with id, name, generation_config, created_at, total_questions, pending_count, approved_count, rejected_count)
- DELETE /api/projects/{id}/test-sets/{test_set_id} — delete (409 if referenced by experiments)

**Questions:**
- GET /api/projects/{id}/test-sets/{test_set_id}/questions — list (query param: ?status=pending|approved|rejected|edited)
- GET /api/projects/{id}/test-sets/{test_set_id}/summary — stats (total, pending, approved, rejected, edited, completion_pct)

**Constraints:**
- testset_size: 1-100
- num_personas: 1-10
- Max 500 chunks per generation
- Generation can take significant time (LLM calls)

## AC-1: Test Set Generation Form
```gherkin
Given the Test page with a project that has chunk configs
When the user fills in the generation form
Then they can select a chunk config from a dropdown
And set a test set name (optional, auto-generated if blank)
And set testset_size (1-100, default 10)
And set num_personas (1-10, default 3)
And toggle use_personas on/off
And client-side validation prevents submission with out-of-range values (size/personas) <!-- audit-added -->
And click "Generate" to call POST /api/projects/{id}/test-sets
And the Generate button is disabled while generation is in progress
And a loading indicator shows during generation (can take minutes)
And if the request exceeds a reasonable timeout (~5min), a timeout message is shown rather than hanging indefinitely <!-- audit-added -->
And on success, the test set list refreshes to show the new set
And on error, a descriptive error message is shown (422 for no chunks, 429 for rate limit, timeout) <!-- audit-added: explicit error types -->
```

## AC-2: Test Set List
```gherkin
Given the Test page
When test sets exist for the project
Then they appear in a list showing: name, question count, status breakdown (pending/approved/rejected), created date
And each test set has a "Delete" button with confirmation
And delete calls DELETE /api/projects/{id}/test-sets/{id}
And if delete fails with 409 (referenced by experiments), the error is shown to the user
And the list refreshes after create/delete
And an empty state message is shown when no test sets exist
```

## AC-3: Question List View
```gherkin
Given a test set in the list
When the user clicks on a test set (or clicks "View Questions")
Then the questions for that test set are displayed
And a header shows the test set name and generation config context <!-- audit-added -->
And each question shows: question text, reference answer (truncated), question type badge, persona badge, status badge
And questions can be filtered by status (all, pending, approved, rejected, edited)
And the status filter calls GET /api/projects/{id}/test-sets/{id}/questions?status=X
And the summary bar always shows unfiltered totals regardless of active filter <!-- audit-added -->
And a summary bar shows total/pending/approved/rejected/edited counts
```
