## Goal
Build a results dashboard for completed experiments — aggregate metric score bars, per-question results table with expandable rows showing response text and retrieved contexts, and metric visualization.

## Purpose
After running an experiment (12-01), users need to see how their RAG pipeline performed. This is the core analysis view that drives iteration decisions.

## Output
- ExperimentResults component with aggregate metrics display
- QuestionResultRow component with expandable detail view
- Integration into ExperimentPage for completed experiments

<acceptance_criteria>

## AC-1: Aggregate Metrics Display
```gherkin
Given a completed experiment is selected from the list
When the results dashboard loads
Then aggregate metrics from experiment.aggregate_metrics are displayed as labeled score bars (0-1 scale) with numeric values
And an overall score (average of all metrics) is shown prominently
```

## AC-2: Per-Question Results Table
```gherkin
Given results are loaded for a completed experiment
When the user views the results dashboard
Then each question is shown as a row with: question text (truncated), question_type badge, per-metric scores as mini score bars
And clicking a row expands it to show: full question, reference answer, model response, retrieved contexts, and all metric values
```

## AC-3: Empty and Loading States
```gherkin
Given the user selects a completed experiment
When results are being fetched
Then a loading skeleton is shown
And if no results exist, an empty state message is displayed
```

## AC-4: API Error Handling <!-- audit-added -->
```gherkin
Given the user selects a completed experiment
When the results API call fails (network error, 404, 500)
Then an error message is displayed with the failure reason
And a retry button is available to re-attempt the fetch
And the UI does not crash or hang
```

## AC-5: Null Aggregate Metrics <!-- audit-added -->
```gherkin
Given a completed experiment has results but aggregate_metrics is null
When the results dashboard loads
Then a "Metrics not computed" message is shown in the aggregate section
And per-question results still render normally below
```
