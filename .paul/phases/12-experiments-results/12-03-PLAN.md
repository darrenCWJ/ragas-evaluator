<objective>
## Goal
Add side-by-side experiment comparison with delta visualization and a historical experiment timeline to the Experiment page.

## Purpose
Users need to compare multiple experiments against the same test set to see which RAG configuration performs best, and track performance trends over time. This completes the Analyze stage of the v0.2 frontend.

## Output
- ExperimentCompare component: multi-select experiments, side-by-side aggregate metrics with delta indicators, per-question comparison table
- ExperimentHistory component: chronological timeline of completed experiments with overall scores and sparkline-style trend
- API client additions: `compareExperiments()` and `fetchExperimentHistory()` functions + types
</objective>

## AC-1: Multi-Experiment Comparison Selection
```gherkin
Given the user has 2+ completed experiments using the same test set
When the user toggles checkboxes on completed experiment cards and clicks "Compare Selected"
Then the comparison view loads showing side-by-side aggregate metrics for all selected experiments
```

## AC-2: Comparison Aggregate Metrics with Delta
```gherkin
Given a comparison view with 2-5 experiments loaded
When the user views the aggregate metrics section
Then each metric shows a bar per experiment, color-coded, with the experiment name label
And the best score per metric is highlighted
```

## AC-3: Per-Question Comparison Table
```gherkin
Given a comparison view is loaded
When the user scrolls to the per-question section
Then each question shows a row with metric scores per experiment side-by-side
And the user can expand a question to see response text and contexts per experiment
```

## AC-4: History Timeline
```gherkin
Given the project has completed experiments
When the user views the history section on the Experiment page
Then completed experiments are shown in reverse chronological order
And each entry shows name, overall score badge, completion date, and RAG config name
```

## AC-5: Empty and Error States
```gherkin
Given no completed experiments exist (or comparison fails)
When the user views comparison or history
Then appropriate empty state messages or error messages with retry are shown
```

## AC-6: Compare API Error Differentiation <!-- audit-added -->
```gherkin
Given the user selects experiments that use different test sets or are not completed
When the comparison API returns 409
Then a specific message is shown (e.g., "Selected experiments must use the same test set" or "All experiments must be completed") with no retry button — only a dismiss/close action

Given the user selects experiments producing too many results
When the comparison API returns 413
Then a specific message is shown ("Too many results — select fewer experiments") with no retry button
```

## AC-7: Client-Side Selection Bounds <!-- audit-added -->
```gherkin
Given the user is selecting experiments for comparison
When fewer than 2 or more than 5 experiments are checked
Then the "Compare Selected" button is disabled with appropriate tooltip text
```
